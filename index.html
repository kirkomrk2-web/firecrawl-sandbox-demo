Node.js execution

Execute arbitrary code inside the sandbox.

The browserExecute endpoint streams Node.js, Python, or bash into your session and returns stdout, stderr, and the last expression value. Press play to replay the sample run below.

sample-run.js · sandbox @ — replay · baked 
// npm install @mendable/firecrawl-js
import Firecrawl from '@mendable/firecrawl-js';

const firecrawl = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

// 1. Launch a disposable browser session
const session = await firecrawl.browser({ ttl: 600, activityTtl: 300 });
console.log('session:', session.id);
console.log('live view:', session.liveViewUrl);

// 2. Navigate + extract top 5 HN headlines
// Sandbox runs code in a Node REPL: the last expression becomes `result`.
const extract = await firecrawl.browserExecute(session.id, {
  language: 'node',
  code: `
    await page.goto('https://news.ycombinator.com', { waitUntil: 'domcontentloaded' })
    await page.waitForSelector('tr.athing')
    const hn = await page.$$eval('tr.athing', rows =>
      rows.slice(0, 5).map((row, i) => {
        const link   = row.querySelector('.titleline > a')
        const sub    = row.nextElementSibling
        const domain = row.querySelector('.sitestr')?.innerText ?? null
        const points = parseInt(sub.querySelector('.score')?.innerText) || 0
        const by     = sub.querySelector('.hnuser')?.innerText ?? null
        const cmts   = [...sub.querySelectorAll('a')]
                         .map(a => a.innerText)
                         .find(t => /comment/.test(t)) ?? '0'
        const age    = sub.querySelector('.age')?.innerText ?? null
        return { rank: i+1, title: link.innerText, url: link.href,
                 domain, points, author: by,
                 comments: parseInt(cmts) || 0, age }
      })
    )
    JSON.stringify(hn)   // last expression → extract.result
  `,
});

const stories = JSON.parse(extract.result);
console.log(stories.length, 'stories');

// 3. Always clean up
await firecrawl.deleteBrowser(session.id);
Capabilities

What a single sandbox session can do.

Each session is a fully isolated Chromium container with Playwright and agent-browser pre-installed. Scale from one browser to hundreds of parallel sessions without touching local infrastructure.

Navigate & interact

Full Playwright API: goto, click, type, waitForSelector, screenshots, and downloads — all driven from your own code.

Execute node · python · bash

Send any snippet via POST /v2/browser/:id/execute. Get back stdout, stderr, result, and exitCode.

Isolated & disposable

Every session runs in a fresh container. No filesystem access, no local network reach, no prompt-injection blast radius.

Live view + CDP

Watch sessions in real time via an embeddable stream, or connect your own Playwright over wss:// CDP for full local control.

Persistent profiles

Give a session a named profile to carry cookies, auth, and localStorage across runs — perfect for logged-in workflows.

agent-browser CLI

Skip Playwright entirely. Send bash commands like "click @e5" or "snapshot" — 60+ commands pre-installed.

Integrate

Launch a session in your stack.

Copy-paste equivalents for the Node SDK, Python SDK, the CLI, and raw cURL.
